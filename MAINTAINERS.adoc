This is a sole-maintainer project. All responsibilities below rest with the
lead maintainer.

== Areas of Responsibility

* *Core interface* (`Base`, `Warrant`, `Access`) — the indexed endofunctor
  `E : K -> Set ℓ -> Set ℓ` and its lawful / factive / belief / return
  variants; keeping the base interface honestly `map`-only.
* *Echo + Surreal bridges* (`EchoBridge`, `SurrealBridge`) — the min-plus
  graded loss/residue `Echo r A`, kept distinct from `E κ A`, and the
  graded surreal access tower. Cross-checked against sibling `echo-types`.
* *Proof transport* (`ProofTransport`, `ProofTransportExample`) — the
  standpoint-indexed transport of attestations across trust boundaries
  (statuses, transmit/verify, no-smuggling), plus its a2ml/k9 engineering
  rendering under `.machine_readable/proof-transport/`.
* *Build + honesty invariants* — that `just check` stays green
  (`agda --no-libraries -i src src/EpistemicTypes/All.agda`, Agda 2.8.0),
  under `--safe --without-K`, with no postulates and no standard library.
* *Documentation* — `readme.adoc`, `explainme.adoc`, and the machine-readable
  6a2 / contractile descriptors.

== Contribution Process

Contributions are welcome via:

1. *Issues* — report errors, suggest formers or lemmas, ask questions.
2. *Pull Requests* — submit changes for review.

Every change must keep `just check` green and preserve the honesty
invariants (no postulates, no stdlib, `--safe --without-K`). All
contributions are reviewed by the lead maintainer.

== Decision Making

* Routine fixes and small lemmas: decided by the maintainer.
* New type formers or interface changes: discussed in an issue first,
  with attention to whether they belong here or upstream in `echo-types`.
* Anything weakening the honesty invariants: out of scope by default.
